# 01. Introduction — Remapping Startup for the AI-Native Era

In May 2026, Anthropic published *The Founder's Playbook: Building an AI-Native Startup*. It remaps the four classic startup stages — Idea, MVP, Launch, Scale — for the reality of 2026, when one founder + Claude Code can ship what used to take a three-person team three months.

This course distills the playbook into 6 modules for quick reference. **All original content is © Anthropic.** This course serves as a study companion and Chinese-language gateway.

## Why a New Playbook

The old startup playbook (Lean Startup, YC) assumed:
- Engineers are scarce → keep the MVP small
- Customer discovery requires legwork → long sales cycles
- Product iteration needs an engineering team → hiring is the bottleneck

In 2026, AI broke all three assumptions:

| Old default | AI-native era |
|-------------|---------------|
| Engineers scarce → "smallest" MVP | Engineering nearly free → "most valuable" MVP |
| Customer research is slow | Research can run in parallel batches |
| Hiring is the bottleneck | Hiring agents is the new bottleneck |
| Revenue before scale | **Revenue still comes before scale** (unchanged) |

The last point holds — but because the first three changed, **the path to revenue is completely different**. That's what the playbook addresses.

## The Four-Stage Map

| Stage | Core question | Exit criteria |
|-------|--------------|---------------|
| **Idea** | Does the problem really exist? | 10+ specific paying-ready customers |
| **MVP** | Can I build a working version cheaply? | First customers can complete the core flow |
| **Launch** | Real PMF or early hype? | Retention/word-of-mouth stable 4+ weeks |
| **Scale** | Solo to team/agent-driven ops | Key flows run when founder is absent |

Each stage gets **Goals**, **Exit criteria**, **Common failure modes**, and **AI-powered exercises** in modules 02–05.

## Tool Matrix: Chat / Cowork / Code

| Stage | Primary tool | Example |
|-------|------------|---------|
| Idea | **Chat** (Claude.ai) | Customer interviews, competitive maps, hypotheses |
| MVP | **Claude Code** | Generate MVP code, fix bugs, write tests |
| Launch | **Cowork** + Code | Marketing flows, content, feedback triage |
| Scale | **Cowork** multi-agent | Replace founder attention with agentic workflow |

> Coffee CLI users: **Claude Code** in the table above is a CLI tool — open it as a new tab in the Coffee CLI desktop app. **Cowork** and **Chat** are web apps, used on [claude.ai](https://claude.ai).
