# 04. Launch 阶段 — 区分 PMF 与早期幻觉

> **退出标准**：核心指标（留存 / 推荐 / 付费转化）稳定 4 周以上不下滑，并且你能说清楚"为什么"。

发布阶段是创业最容易自我欺骗的阶段。Twitter 上的好评、Product Hunt 的当日榜首、媒体的报道——这些都不是 PMF。它们是**烟花**。区分烟花和真火，是 Launch 阶段的核心功夫。

## 阶段目标

1. **找到一个有意义的留存信号**（不是 vanity metrics）
2. **建立 agentic 营销/运营工作流**，把创始人从重复劳动里拔出来
3. **判断是真 PMF 还是早期炒作**，决定是 Scale 还是回 MVP
4. **从"我推产品"切到"产品自己长"**

## PMF 衡量框架

### 一个核心问题

Sean Ellis 的经典问题至今有效：

> "如果明天这个产品消失了，你会有多失望？"

四个候选答案：

- 非常失望
- 有点失望
- 不会失望
- 我已经不用了

**40% 以上选"非常失望"** = 触摸到 PMF 的边了。低于这个数，回去打磨。

### 三个不会撒谎的指标

烟花指标：DAU、注册数、发推热度  
**真火指标**：

1. **D7 / D30 留存**：用户 7 天后还在用吗？30 天后呢？
2. **付费转化率**：免费试用 → 付费的比例，连续 4 周稳定 ≥ 5%
3. **自然推荐率**：新用户里有多少是"朋友/同事推荐来的"

**4 周稳定** 比"有过一次峰值"重要 10 倍。

### 不要数 vanity metrics

| 烟花指标 | 真火指标 |
|---------|---------|
| Product Hunt 排名 | 上线 30 天后的 DAU |
| Twitter follower 数 | 邮件订阅 → 注册转化 |
| 媒体报道数量 | 自然搜索流量增长 |
| 当日注册量 | D30 留存 |

## Agentic 运营工作流

发布之后，创始人的注意力会被以下事情吃光：

- 回客户邮件 / Discord
- 写营销内容（推文、博客、case study）
- 跑数据分析
- 处理客户反馈分类

**把这些都交给 agent**，留出 80% 的时间做"只有创始人能做"的事（产品方向、关键招聘、关键客户）。

### 一个典型的 agentic 营销栈

| 工作 | 工具 | 谁做 |
|------|------|------|
| 客户反馈分类 | Claude + Linear/Notion API | Agent 全自动 |
| 周报数据汇总 | Claude Code + dbt | Agent 全自动，创始人审 |
| 内容初稿（推文、博客） | Cowork | Agent 写，创始人改 |
| 客户邮件回复 | Claude.ai 模板 + 人工发 | Agent 起草，创始人按钮发 |
| 关键客户跟进 | **创始人亲自** | 不要让 agent 碰 |

最后一条划线：**关键客户必须创始人亲自跟**。这是 agent 不能替代的。

## 常见失败模式

| 失败模式 | 原因 | 怎么破 |
|---------|------|--------|
| **把烟花当成真火** | 上线当周数据爆炸，以为成了 | 等满 4 周再判断 |
| **过早烧钱做付费投放** | "再加一倍预算流量会更好" | 先做到自然增长 ≥ 5%/周再投钱 |
| **创始人继续手动回复一切** | "客户体验" | 一天投入 ≤ 2 小时手动，剩下让 agent 干 |
| **PMF 没到就开始招人** | "再来 3 个工程师我们就赢了" | 没 PMF 招人 = 放大错误 |
| **以为 PMF 是一个点** | "我们找到 PMF 了" | PMF 是一段持续状态，会丢，要守 |

## 退出 Launch 阶段的检查清单

- [ ] D30 留存 ≥ 30%（B2B SaaS）或 ≥ 40%（C 端高频）
- [ ] 付费转化连续 4 周 ≥ 5%
- [ ] 至少 20% 新用户来自自然推荐
- [ ] 创始人每天用在重复劳动上的时间 ≤ 2 小时
- [ ] 你能用一句话说清"为什么用户留下来"

如果"为什么留下来"答不上，那不是真 PMF，是运气。继续观察。
