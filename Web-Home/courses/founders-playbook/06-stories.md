# 06. 创始人案例 — 六家公司的实战故事

Anthropic 在《The Founder's Playbook》里挑了 6 家正在用 Claude 跑 AI 原生模式的早期公司，作为案例研究。它们覆盖了不同行业（医疗、企业工具、消费应用、agent 基础设施），共同点是：**小团队、AI 重度使用、营收驱动**。

> 以下案例信息均基于 Anthropic 官方手册公开内容整理。每家公司的详细故事请以官方原文为准。

## Ambral

**行业**：企业应用 / 自动化  
**模式**：小团队 + Claude Code 重度使用，把企业流程自动化做到"按结果收费"

**关键看点**：
- 从一开始就用 AI 写代码，工程人头压到最低
- 销售采用"先免费跑出 ROI、再签合同"的模式
- 客户成功大量 agent 化

## Anything

**行业**：消费应用 / 创意工具  
**模式**：把 AI 能力包装成"普通人也能用"的创作工具

**关键看点**：
- UX 优先：把 AI 的"魔法时刻"做到 10 秒内
- 增长靠用户自发分享，几乎不投放
- 创始人继续亲自做产品 + 见用户

## Carta Healthcare

**行业**：医疗 / 临床数据  
**模式**：用 AI 处理临床试验的非结构化数据

**关键看点**：
- 强监管行业 + AI = 极高门槛，护城河天然深
- 销售周期长，但 ACV（年合同价值）极高
- 工程团队小，但合规和领域专家配比高

## HumanLayer

**行业**：Agent 基础设施  
**模式**：给 agent 加"人类审批"层，让自动化系统在关键决策点等人

**关键看点**：
- 卖给"做 agent 的人"——dev tools 玩家
- 完美契合 Scale 阶段的"agent + 人"分工框架
- 创始人本身就是 agent 框架的深度用户

## Vulcan Technologies

**行业**：B2B / 工业自动化  
**模式**：把传统工业软件的"半手工"流程用 AI 重做

**关键看点**：
- 老行业 + AI 是当前最被低估的赛道
- 客户付费意愿高，但教育周期长
- 销售必须创始人亲自做（信任问题）

## 共同模式

把 6 个案例横向对比，能看到几个共同特征：

| 共同点 | 解释 |
|--------|------|
| **团队 < 20 人** | AI 让小团队也能做出过去需要 50 人的事 |
| **创始人 / 工程比 ≥ 1:3** | 不是工程师堆人，是 AI 杠杆 |
| **客户成功 / 销售比 ≥ 1:1** | 销售依然重要，不是只靠产品自传播 |
| **毛利率 > 80%** | AI 带来的成本结构优势 |
| **创始人亲见客户** | 无论团队多大，这条不变 |

## 怎么用这些案例

读案例最容易犯的错：**记住故事忘记原则**。

正确读法：
1. 先读案例，记住"它做对了什么"
2. 回到 02-05 模块，对应到具体阶段
3. 问自己："我现在的阶段，应该模仿它哪一招？"
4. **不要直接复制商业模式**——你不是它
