# 03. MVP 阶段 — 造出"刚刚好"的产品

> **退出标准**：首批客户可以用你的产品完整跑一遍业务流程，不需要你在旁边"手动救场"。

MVP 阶段最大的变化：**写代码不再是瓶颈**。一个创始人配上 Claude Code，一周内可以做出过去三人小队三个月才能做出的功能集。但产能爆炸带来一个新陷阱——**架构和安全债比代码量增长得更快**。

## 阶段目标

1. **造出"端到端"可用的版本**，而不是漂亮的局部 demo
2. **架构选型要为"3 个月后还要改"留余地**，不是"先跑通再说"
3. **从第一天就把安全做对**，因为 AI 生成的代码默认不安全
4. **建立可衡量的客户反馈通路**，准备进入 Launch 阶段

## 架构与范围

### 三条范围红线

AI 代码生成快不代表你应该全做。MVP 阶段死守以下三条：

1. **不做付费/支付**：先用 Stripe Payment Link 或人工开发票，省 1 个月工程量
2. **不做后台管理界面**：用 Retool / Airtable + 数据库直连，省 2 周
3. **不做用户角色权限系统**：所有人都是 admin，等 Launch 后再加

> 这三条加起来能省你 2 个月。把这 2 个月用在客户访谈和迭代上。

### 架构选型的"3 个月规则"

问每一个技术决策：

> "3 个月后我要改这个，代价是多少？"

- 数据库选 Postgres，不是 NoSQL：3 个月后想加复杂查询，Postgres 不会卡你
- 后端用单体，不是微服务：3 个月后想拆服务，单体很容易拆；反过来很难合
- 前端用一个框架，不要混：3 个月后招人，一个框架的人好招

**AI 不会替你做这些选型**，因为它不知道你 3 个月后想干嘛。这是创始人的活。

## 安全实践

AI 生成的代码默认有以下问题：

| 问题 | 表现 | 怎么修 |
|------|------|--------|
| **SQL 注入** | 字符串拼接 SQL | 强制用参数化查询 / ORM |
| **硬编码密钥** | API key 写在代码里 | `.env` + 检查 .gitignore |
| **任意文件读** | path traversal | 白名单 + path.resolve 校验 |
| **CORS 配错** | `Access-Control-Allow-Origin: *` | 显式列域名 |
| **没有 rate limit** | 一个用户能把服务跑挂 | 用 Cloudflare / 框架中间件 |

**在 MVP 阶段就装这些 guardrail**，比上线后修便宜 10 倍。

## 防止技术债爆炸

### 给 Claude Code 立规矩

在项目根目录建 `CLAUDE.md`（或对应 `AGENTS.md`），明确写：

```
- 所有数据库查询用 prepared statement
- 不允许在代码里写硬编码 secret
- 新功能必须有至少 1 个集成测试
- 修改超过 50 行的 PR 必须先写一段设计说明
```

这是给 AI 看的"项目宪法"。Claude Code、Cursor、Aider 都会读它。

### 每周一次"债务清盘"

每周固定 2 小时，让 Claude 跑一次：

> "扫一遍项目，找出：1) 重复了 3 次以上的代码片段；2) 没有测试覆盖的关键路径；3) 不符合 CLAUDE.md 规则的地方。"

把扫出来的前 3 项当周修掉。剩下的可以等。

## 常见失败模式

| 失败模式 | 原因 | 怎么破 |
|---------|------|--------|
| **造成"演示型 MVP"** | 只能演示 happy path，真用就崩 | 强制让 3 个客户独立完整跑一遍 |
| **过早优化性能** | "10 万用户怎么办" | 10 万用户是 Scale 阶段的问题 |
| **AI 写多少接多少** | 不读代码就 commit | 每个 PR 至少看一遍 diff |
| **每个功能都自己造** | "不依赖第三方" | MVP 阶段不存在"过度依赖"，能买就买 |
| **没有客户反馈通路** | 上线后才想起来要反馈 | 第一天就建客户群 / 留邮箱按钮 |

## 退出 MVP 阶段的检查清单

- [ ] 首批 3 个客户能不靠你完整跑一遍核心流程
- [ ] 关键安全检查项都通过（SQL/密钥/CORS/rate limit）
- [ ] 项目根目录有 `CLAUDE.md` 并被实际遵守
- [ ] 你能在 1 周内 ship 一个新功能（不会被技术债拖住）
- [ ] 有渠道能 24 小时内收到客户反馈

如果"首批客户独立跑通"这条没达成，**不要进入 Launch**。继续打磨。
