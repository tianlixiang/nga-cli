# 01. 课程导读 — 用 AI 重画创业地图

2026 年 5 月，Anthropic 发布了《The Founder's Playbook: Building an AI-Native Startup》（《创始人手册：打造 AI 原生创业公司》）。这是一本写给"在 AI 时代从零做产品"的创始人的实战手册，把创业的四个经典阶段——构想（Idea）、最小可行产品（MVP）、发布（Launch）、规模化（Scale）——按 2026 年的现实重新画了一遍。

本课程把这本手册的核心方法整理成 6 个模块，供中文读者快速对照原文使用。**所有原始内容版权归 Anthropic 所有**，本课程仅作为学习导航与中文注解。

## 为什么需要一本"新"创业手册

过去十年的创业方法论（精益创业、Y Combinator 那一套）默认的前提是：
- 工程师稀缺 → MVP 要小、要省人
- 客户访谈要靠双脚 → 销售周期长
- 产品迭代要靠工程团队 → 招人是瓶颈

到了 2026 年，AI 把这三个前提全都打破了：

| 旧默认 | AI 原生时代 |
|--------|------------|
| 工程师稀缺，MVP 要"最小" | 工程产能近乎免费，MVP 要"最值得" |
| 客户访谈靠双脚 | 客户研究可以并行批量做 |
| 招人是瓶颈 | 招人变成"agent 招架"问题 |
| 营收先于规模 | **营收依然先于规模**（这条没变） |

最后一条没变，但因为前三条变了，**到达营收的路径完全不同了**。这就是手册要解决的问题。

## 四阶段地图

| 阶段 | 核心问题 | 退出标准 |
|------|---------|----------|
| **Idea**（构想） | 我要解决的问题真的存在吗？ | 已找到至少 10 个愿意付费的目标客户 |
| **MVP**（最小可行产品） | 我能不能用最低成本造出"管用"的版本？ | 产品能让首批客户完整跑一遍业务流程 |
| **Launch**（发布） | 这是真 PMF 还是早期幻觉？ | 留存与口碑数据稳定 4 周以上 |
| **Scale**（规模化） | 从单兵作战切到团队/agentic 运营 | 关键流程可在创始人不在场时跑通 |

每个阶段，手册都给出：
- **Goals**（目标）
- **Exit criteria**（退出标准）
- **Common failure modes**（常见失败模式）
- **AI-powered exercises**（AI 驱动的练习）

后续 4 个模块（02–05）逐一展开。

## 工具矩阵：Chat / Cowork / Code

Anthropic 在手册中提供了一张"什么阶段用什么工具"的矩阵。简化版如下：

| 阶段 | 主力工具 | 用法举例 |
|------|---------|----------|
| Idea | **Chat**（Claude.ai 对话） | 跑客户访谈脚本、画竞品地图、写问题假设 |
| MVP | **Claude Code** | 直接生成 MVP 代码、修 bug、写测试 |
| Launch | **Cowork** + Code | 跑营销流、内容生成、客户反馈分类 |
| Scale | **Cowork** 多 agent | 把"创始人注意力"换成 agentic workflow |

> Coffee CLI 用户提示：上表里的 **Claude Code** 是命令行工具——在 Coffee CLI 桌面应用里新开一个 Claude Code 标签页就能运行。**Cowork** 和 **Chat** 是网页应用，在 [claude.ai](https://claude.ai) 中使用。

## 如何使用本课程

1. **第一遍快速过**：把 02–05 四个模块的标题和退出标准看一遍，建立阶段感
2. **对照原文**：访问下方"原文与资源"链接下载 Anthropic 官方 PDF
3. **回到自己的项目**：对每个阶段的"失败模式"自检，看自己当前卡在哪里
4. **6 模块案例**：读完后看 06，6 家创业公司的实战故事
